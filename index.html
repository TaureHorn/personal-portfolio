VOLCANOES

A small project built during my first week on The Developer Academy Web Development Bootcamp. Prior to this I had never coded before, beyond changing some values in CSS and a writing couple of <p> and <br> tags in HTML. So I just wrote about something I know well. Volcanoes! I spent most of my MSc studying them so best that I get some use out of that knowledge.

This project took a couple of days to put together and was a good challenge considering I knew nothing about coding a few days prior to starting. Getting to grips the powers of Bootstrap and Flexbox was a both interesting and frustrating experience. Neveretheless, given the time invested and the knowledge gained I'm happy with this project. Sure I could now make something better in half the time, sure it's kinda ugly and, sure there's no interactability or JavaScript to speak of: but for a HTML and CSS newb I'm pretty proud.

I utilised Bootstrap components as much as I could to save time and effort configuring the minutae of CSS. Additionally, I enjoyed experimenting with different ways of achieveing the same effects in this site. For example, I used Bootstrap cards in two different configurations and used figures to control images as well. Just some things to accelerate my learning.

FANTASY BESTIARY QUIZ

A small project built during my second week on The Developer Academy Web Development Bootcamp. This week aimed to introduce the basics of JavaScript. As such we were tasked with writing a quiz. Having rather enjoyed the desing and CSS based aspects a lot in the previous weeks project this is where a lot of my effort was spent. Though that's not to say the JavaScript went ignored.

Design wise, the site was comprised of ~25% preamble story and ~75% quiz. A quick introduction to a character - Volo - and said character grilling you on your ability to identify mythological and fantastical creatures based on a few esoteric descriptions. I had a lot of fun with both the themeing of this project, both aesthetically and conceptually.

As for the JavaScript, as was the learning intention, it all worked out well. The code was simple in both concept and execution. A an array to track the answers, a series of functions to recieve and parse input, an array to track the number of correct answers and finally a big if - else to give an response tailored to how well the user did on the quiz. In hindsight, the code could definitely be refactored down to less than half its length. But, given that this was written a few days into learning JavaScript the fact that the quiz works at all is something of a success.

HACKING TEXT GAME

A small project built during my second week on The Developer Academy Web Development Bootcamp. This weeks learning task was to continue learning the fundamentals of JavaScript, though this time with Object Oriented Programming (OOP) in mind. Thus we were set with a task to write a small text adventure game, where a user can navigate through different areas, and interact with a pre-written environment.

This project was an important less on scale, ambition and hubris. The concept for this project was one that got me very excited. So much so, I made a headstart on it, spending most of the weekend prior to when we were meant to start working on the design elements. I had the idea to abstract the concept of navigating through rooms and interacting with items to the user naviagates through directories and interacts with files. In essence it was to be a simplified version of interacting with a linux system in a bash terminal. In function however, it was not anything. Most of the actual game interaction elements did not function within the time that I had. Though, much of the interface was navigable with different display toggled subsections and information therein. I even implemented to system to toggle between one of six themes.

This project ultimately failed at the time, not for a lack of effort, but for an error in scope. I had based the idea off another game called HackNet which ultimately likely too much more time than the week that I had, and was surely made by much more experienced programmers than I at three weeks into a bootcamp. In hindsight, it was far too much work to try to achieve what I had set out to in the given timeframe and with respect to my level of knowledge and experience.

Despite the ultimate failure of this project to deliver on a game as was the original intent it was still successful in a few ways.

  1. A learnt a lot about assessing the scope and feasibility of a project while in the plannng phase.
  2. My skill and experience with JavaScript still definitely improved
  3. While I was unable to ultimately execute the OOP method, I still learnt a lot about how OOP is supposed to work.
  4. My proficiency with other tools increased dramatically. Particularly with Flexbox and CSS.

I still like the idea of this project and aim to revisit it in the future to get it to a working state as originally intended.

DUNGEONS AND DRAGONS

Perhaps you are reading this and thinking -"This is supposed to be a professional portfolio for serious work. What is this idiot doing bragging about gathering with nerds and pretending to stab goblins in his leisure!-

Were it just that I play Dungeons and Dragons it would be a reasonable - if mean spirited - thing to think. The case however is a little more complicated and if you stick with me, I might be able to explain its relevance to my skills and as creative and commerically useful person.

I've played DnD for about the last 9 years. Starting just before the current 5th Ed came out - if you are familiar. For the past ~4 years of that time, I have been running my own game as a so called Dungeon Master, or DM. This role is absolutely integral to the funcitoning of any game of DnD. As a DM, it is my responsiblity to provide the players with a world, character and plot-lines to interact with and adventure in. And yes, sometimes there are goblins to stab. This game I have been running is what is called 'homebrew', meaning entirely custom and made by me. That means the world, all the characters, all the maps and all the plotlines are written by me. No small feat to keep a group of DnD nerds entertained on a weekly basis for that length of time I assure you. -Tens of thousands of words of text, hundreds of self-made maps e.t.c

Now, to the relevance to my skills. I run this game online and the players interact with the world with the help of maps that I make. These maps started out very simply, but slowly over the course of the last four years I have taught myself to draw and made the consious decision to put effort into the creativity it requires to make these maps. LEARNING LINUX

When the country went into lockdown and work from home became a very popular thing I, like so many others, had to adjust. For a sense of psychological separation, I dual-booted my computer. I was using Windows at the time but installed regular old Ubuntu alongside for work. That was not my first experience using a linux OS, but it was certainly the first time I had attempted to use one for real work and for a signifcant length of time. As luck would have it, I decided I much preferred Ubuntu to Windows and soon completely transitioned to Ubuntu. KDE more specifically if you're aware of the significance.

Since switching over to linux I've been slowly teaching myself to use a bash terminal. Of course I'm no sysadmin command line wizard but I can cd, ls, cat, sudo, mv, cp, ssh, find, e.t.c with confidence. So far in my progress of learning web development, these skills have been a godsend when working with git and GitHub. Further, they were essential for a side project of setting up a Raspberry Pi as a Network Attatched Storage (NAS) device.

Of course, I can't easily exemplify those skills, so in their stead, I can show something else. Among other things, one stereotype of some linux users is their obssesion with customisation and showing off that customisation. In that vein, I customised the appearance of much of my KDE desktop environment to exactly suit my aesthetic preferences.
